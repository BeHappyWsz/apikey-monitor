# 实施计划

1. 调整 `core/probe.py`：能力字段只由对应协议本次探测的 `status == "up"` 决定；后台健康检查完整探测所有已注册协议并覆盖旧能力。
2. 保持健康检查不执行模型调用，仅刷新连通状态和协议能力。
3. 调整 `static/js/cards.js`：无确认能力时显示“未确认”，多种成功能力继续全部展示。
4. 更新后端测试，覆盖 401/403 不确认能力、监测失败清除能力、监测发现全部协议。
5. 运行 Python 单元测试、Node 语法检查和前端状态测试。

## 实施结果

- 协议能力改为仅由本次 `up` 探测结果确认。
- 手动检测与后台监测均全量探测并覆盖旧能力。
- 面板在非 `up` 状态隐藏能力标签，无能力时显示“未确认”。
- Python 86 项测试、Node 15 项测试和全部前端语法检查通过。
