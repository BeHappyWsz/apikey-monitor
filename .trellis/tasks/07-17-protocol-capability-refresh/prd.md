# 修正协议能力检测与更新

## Goal

让密钥实例面板中的协议能力只反映最近一次成功探测的协议，并在后续检测或监测失败时清除过期能力。

## Requirements

- OpenAI 和 Anthropic 能力分别依据对应探测结果判断。
- 只有 `status=up` 的协议可以写入能力字段。
- 支持几种协议就展示几种，不设置优先级。
- 手动检测和后台监测都必须覆盖更新能力字段。
- 待检测或没有成功协议时，面板显示“未确认”。
- 不新增数据库字段，不改变模型检测策略。

## Acceptance Criteria

- [ ] OpenAI 单独成功时只展示 OpenAI。
- [ ] Anthropic 单独成功时只展示 Anthropic。
- [ ] 两种协议均成功时展示两种协议。
- [ ] 401/403、限流、异常和超时不确认协议能力。
- [ ] 后续检测失败会清除以前保存的对应能力。
- [ ] 后台监测可刷新全部协议能力，而不是永久沿用历史结果。
- [ ] 后端测试和前端语法检查通过。
