# API Key 配置与监测项目设计

## 背景

当前项目已经具备本地 Web 服务、SQLite 存储、粘贴解析、协议判别、后台监测和基础前端管理能力。下一步目标是补齐项目文档，并完善管理闭环，使用户可以通过页面完成常见运维动作。

## 范围

本轮包含：

- 生成使用者文档 `README.md`。
- 生成开发者设计文档 `docs/design.md`。
- 生成 HTTP API 文档 `docs/api.md`。
- 前端增加单条编辑能力。
- 前端 API 请求增加错误处理。
- 列表中 API Key 使用脱敏显示。
- 后端增加基础输入校验，避免空值和异常字段写入。

本轮不包含：

- 多用户登录和权限体系。
- 云端部署。
- API Key 加密存储。
- 自动备份和恢复。

## 设计

沿用现有零依赖架构，不引入外部 Python 包或前端构建链。`app.py` 继续作为本地 HTTP 服务入口；`core.py` 保持解析、检测和导出职责；`db.py` 处理 SQLite；`monitor.py` 处理后台调度；`static/` 负责页面交互。

新增编辑能力通过现有 `PUT /api/keys/{id}` 完成。前端提供编辑弹窗，支持名称、Base URL、API Key、备注、是否监测和单条检测间隔。保存后刷新列表；用户如需立即重新判别，可以使用“检测”按钮。

API 错误处理统一在前端 `api()` 方法中完成：HTTP 非 2xx、返回 `error` 字段、JSON 解析失败均展示 toast，并抛出异常阻断后续误操作。

## 验证

- 运行 Python 编译检查，确认语法有效。
- 启动本地服务后可以打开页面。
- 粘贴导入、列表刷新、编辑保存、单条检测、批量检测、导出配置均保持可用。
